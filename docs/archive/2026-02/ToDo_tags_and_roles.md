### Бриф для реализации системы ролей и тегов в RepSync

Этот бриф описывает план внедрения системы ролей пользователей (baseTags в профиле, memberRoles в группах, assignments в сетлистах) и тегов для песен (arbitrary tags с облаком для поиска/импорта CSV) в приложении RepSync (Flutter 3+, Riverpod 2+, Hive, Firestore). Фокус на гибкости для сборных концертов, автозаполнении ролей и использовании тегов для экспорта списков участников.

**Цели проекта**:
- Внедрить базовые теги ролей в профиле с pull в банды/песни/сетлисты.
- Добавить overrides ролей для гостей/тональности в сетлистах.
- Внедрить произвольные теги для песен с аггрегацией в облако тегов.
- Обеспечить синхронизацию: Hive для оффлайн, Firestore для коллаборации.
- Минимизировать зависимости: Использовать textfield_tags для ввода тегов.
- Тестирование: Unit для ноттификаторов, integration для синка, manual UI.
- Риски: Конфликты синка (использовать lastUpdated); производительность облака тегов (ограничить сканирование).
- Метрики успеха: Роли/теги сохраняются оффлайн; поиск по тегам работает; экспорт включает участников.
- Timeline: 4-6 дней, ежедневные commits в GitHub.

**План работы**: Разделено на спринты по возрастающей сложности. Каждый — MVP с тестами. В конце — однострочные TODO.
