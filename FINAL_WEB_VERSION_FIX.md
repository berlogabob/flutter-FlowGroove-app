# Web Version Display Fix - Direct version.json Fetch

## Problem

Web version was showing `0.13.0` without build number even though:
- ✅ `version.json` had correct buildNumber: `146`
- ✅ Profile screen code was combining version+buildNumber
- ✅ Tested in incognito + Tor (not a cache issue)

## Root Cause

**`package_info_plus` on web doesn't correctly read buildNumber from version.json**

The package reads the file but returns:
```dart
packageInfo.version      // "0.13.0" ✅
packageInfo.buildNumber  // "" or "1" ❌ (incorrect!)
```

## Solution

**Directly fetch `version.json` on web instead of using `package_info_plus`**

### Implementation

**File:** `lib/screens/profile_screen.dart`

Added web-specific version loading:

```dart
Future<void> _loadVersionInfo() async {
  // On web, directly fetch version.json
  if (kIsWeb) {
    final response = await http.get(Uri.parse('version.json'));
    final data = jsonDecode(response.body);
    final version = data['version'];        // "0.13.0"
    final buildNumber = data['buildNumber']; // "146"
    
    _version = '$version+$buildNumber';     // "0.13.0+146" ✅
  }
  
  // Fallback to PackageInfo for other platforms
  final packageInfo = await PackageInfo.fromPlatform();
  // ... rest of code
}
```

### Why This Works

1. **Direct HTTP fetch** bypasses `package_info_plus` entirely
2. **Reads raw JSON** from `version.json`
3. **Correctly parses** both version and buildNumber
4. **Combines them** for display: `version+buildNumber`

## version.json Format

The fix expects this format:

```json
{
  "version": "0.13.1",
  "buildNumber": "146",
  "buildDate": "2026-03-11T16:55:27Z"
}
```

Generated by `scripts/fix-version.sh`:
```bash
✅ version.json fixed:
   Version: 0.13.1+146
   Build Date: 2026-03-11T16:55:27Z
```
